WORKFLOWS
═════════════════════════════════════════════════════════════════════════════

Workflow 1: User Returns Book Late
  1. User returns book 3 days late
  2. Backend creates penalty (€1.50)
  3. User sees in PenaltiesListScreen
  4. Admin marks "Payée" in AdminPenaltiesScreen
  5. Status changes to PAID
  6. User can borrow again

Workflow 2: User Blocked by High Penalties
  1. User accumulates €150 in penalties
  2. Dashboard shows RED PenaltyBadge
  3. User tries to borrow → 403 FORBIDDEN
  4. User sees RED PenaltyAlert
  5. Admin marks €100 payment → total drops to €50
  6. User can borrow again (< €100)

Workflow 3: Admin Cancels Penalty
  1. Admin opens AdminPenaltiesScreen
  2. Selects penalty → clicks "Annuler"
  3. Enters reason: "User hospitalisé"
  4. Confirms → penalty status = CANCELLED
  5. User sees updated status in their view
